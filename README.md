# StayLeased

A multi-tenant property management platform with functional parity to Entrata's suite — property operations, resident experience, accounting, payments, utilities, insurance, revenue intelligence, BI, communications, procurement, an AI-agent layer, and vertical modes — built as **original work** with every external rail (payments, screening bureaus, SMS/email, bank feeds, meters, insurers, LLMs) implemented as a **deterministic simulator** behind a provider interface, so the whole platform runs end-to-end in a sandbox with no real money and no real consumer data.

## Working model — real customers, not just the demo

The platform now runs demo and LIVE orgs side by side. `/signup`
(invite-code-gated via `STAYLEASED_SIGNUP_CODE`) creates a real, empty
company on the real calendar; `/welcome` walks it to go-live; and the
**Migration Center** (`/setup/import`) moves a portfolio in from anywhere:

- **Rent roll / any spreadsheet** (.xlsx/.csv/.tsv — Buildium, AppFolio,
  Yardi, Rent Manager, TenantCloud exports auto-detected; any other columns
  auto-mapped by synonyms + optional AI assist; you confirm before apply).
  One file creates properties, floorplans, units, residents, leases,
  deposits held, and balances owed.
- **Signed lease PDFs** — extracted locally (typed PDFs) or by live Claude
  (scans) into editable drafts with confidence flags.
- **Vendors, extra residents, opening balances, opening bank balance.**

Conversion accounting is done properly: migrated balances post as
`opening_balance` charges against `3030 Opening Balance Equity`, deposits
held post `1020/2100` on both bases, and each migrated lease carries a
`billing_start_date` so recurring rent never double-bills a month the prior
system already charged. Live orgs never run the demo world's simulators
(fake leads/bank feeds/meter reads), can't open the time machine, and issue
one-time passwords instead of `demo1234`.

## Run it

Requires Node ≥ 22.11. The runtime uses only Node built-ins; dev tooling (typescript, playwright) and pdf-lib come from npm.

Not a terminal person? Double-click **Start-StayLeased.bat** (Windows) or **Start-StayLeased.command** (Mac) — see **HOW-TO-RUN.txt**.

```bash
npm install
npm run seed     # build the deterministic demo world (idempotent; -- --reset rebuilds)
npm run dev      # → http://localhost:3000
```

**Live AI (optional).** The AI layer runs in deterministic **demo** mode out of
the box. Set `ANTHROPIC_API_KEY` (and optionally `STAYLEASED_AI_MODEL`, default
`claude-opus-4-8`) to switch the brain to **live** — a raw-HTTPS Anthropic
adapter with per-call/daily token caps, response caching, and a seamless
fallback to the deterministic brain if the key is missing or a call fails. The
active brain (Live/Demo) shows in **Reports → AI Activity**, which also has a
global **kill switch**. Config still accepts the legacy `ORIEL_*` env names as
fallback aliases.

Sign in as any persona — every demo account uses password **`demo1234`** (full table in `docs/demo-logins.md`, generated by the seed):

## Architecture in one paragraph

Modular monolith on Node built-ins: a small server-rendered web framework (`src/lib/http.ts`, `src/lib/html.ts`), `node:sqlite` with a Postgres-compatible schema (`src/db/schema.sql`), and per-module folders under `src/modules/` whose services contain all business logic (UI never touches SQL directly... services do). Every financial event posts through the double-entry accounting core via posting rules; a domain-event log drives cross-module reactions, webhooks, and the AI layer; every mutation is audited. External dependencies are TypeScript interfaces in `src/lib/sim/` with deterministic, seedable simulators as the default implementation. The demo tour lives at the bottom of this README (added at Phase 18).

## The 15-minute demo tour

A scripted click-through of the whole platform against the seeded world. Fresh
`npm run seed -- --reset` first; keep two browser windows handy (staff +
resident). Every step below is verified by the e2e suite.

**1. The money is real (2 min).** Sign in as `accountant@summitridge.demo`.
Open **Money → Accounting**: trial balance is non-trivial (23k+ journal
entries over 14 months) and `/gl/invariants` shows all five §9 invariants
green — AR ties to the ledgers, debits equal credits, both bases balance.
Open **Banking**: 42 account-months reconciled to $0.00; open the Summit Ridge
operating account's July workbench, click **Auto-match**, watch the
difference hit $0.00, and **Complete**. Open **Periods**: June is closed —
try posting a June journal entry from `/gl/journal` and get blocked.

**2. A resident's world (2 min).** In a second window (or phone-sized
viewport) sign in as `maya.torres@mail.demo`. Pay part of her balance with
the saved ACH method; file a maintenance request with a photo ("disposal is
stuck and just hums" is a fun one — the AI will offer the reset-button tip);
check **Lease** for her renewal offer, insurance, usage-vs-community chart
and communication preferences.

**3. Ops on the staff side (2 min).** As `manager@summitridge.demo`: the
**Inbox** shows Maya's renewal counter-question and Derrick's promise-to-pay
(assigned to you, with the internal note). **Work orders** has the new
request; **Delinquency** shows Derrick's household aged into 61–90 with his
payment plan; open his workbench page — notes, promise, plan, dunning, the
AI outreach button.

**4. Leasing front door (2 min).** Open the public site `/p/summit-ridge`
(no login): live pricing, real availability. Book a tour — then as
`agent@summitridge.demo` see the lead + tour in **Leads** with the follow-up
cadence. Open Alicia Nguyen's guest card: quotes, tour history, and the
**✨ Draft AI reply** button.

**5. The day the platform runs itself (3 min).** As
`admin@summitridge.demo` open **Developer → Simulator console** and advance
**+1 day**. Then look at what happened: **Developer → Message console**
(receipts, reminders, the scheduled autopay campaign, the daily delinquency
report with its CSV attachment), **AI Activity** (call analyses, any
autonomous after-hours bookings at Cardinal — every action with input,
output, confidence and dial), **Reports → Saved** (the daily snapshot ran).
ACH payments from step 2 settle after +3 days.

**6. Revenue intelligence (1.5 min).** As `regional@summitridge.demo` open
**Pricing**: 40+ recommendations, each explaining its factors (exposure vs
7% target, days vacant, season, comp positioning, the ±5% guardrail).
Override one with a reason — then reload `/p/summit-ridge` and watch the
public "from" price move instantly. **Term rates** shows expiration
smoothing (heavy months priced up); **Renewal batch** prices Foundry's
expirations live, capped at the org's 8%.

**7. Reports that reproduce history (1.5 min).** Open **Reports** (50-report
catalog). Run **Rent Roll** for Summit Ridge with the date set six months
back — row counts and totals reproduce January exactly, including residents
who have since moved out. Group the **Delinquency/Aged** report by property;
export the PDF. Open **My dashboard** — the exec layout — and customize it.

**8. Verticals + AI supervision (1.5 min).** **Student housing**: Cardinal's
bed board (75% fall pre-lease, pacing curve, roommate matching); assign a
bed and see an individual-liability lease appear. **Affordable**: Foundry's
LIHTC grid — rent vs limit headroom per unit, the overdue recert, the
waitlist with a documented skip. **AI Activity → Autonomy dials**: the
supervision model and the hard guardrails, in plain language.

**9. Ask StayLeased (30 sec).** Top bar → **✨ Ask StayLeased**: "delinquency over
$500 at Summit Ridge", "which units turn this month", "occupancy at
Foundry" — live numbers, drill-through links, and the ask itself lands in
the audit trail.

That's the platform: one seeded world, everything connected, every number
traceable to the ledger underneath it.
